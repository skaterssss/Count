## Project Overview

"Telspel met Dieren" (Animal Counting Game) is a Dutch-language children's educational web game. Children click numbered buttons 1–10 in sequence; each correct click progressively reveals a randomly chosen animal image.

## Running Locally

No build step required. Serve as static files:

```bash
python -m http.server 8000
# or
npx http-server
```

Then open `http://localhost:8000` in a browser.

## Architecture

The project is three files: `index.html`, `script.js`, `style.css`.

**`script.js`** owns all game logic via global state variables (`currentNumber`, `correctAnswers`, `currentAnimal`, `imageLoaded`). Key functions:

- `initGame()` — resets state, selects a random animal, preloads its Unsplash image (falling back to an emoji if it fails), and calls `createNumberButtons()`
- `createNumberButtons()` — generates 10 buttons shuffled with Fisher-Yates
- `handleNumberClick()` — routes to `handleCorrectAnswer()` or `handleWrongAnswer()`
- `revealAnimal()` — progressively uncovers the animal image by manipulating `clipPath` on `.animal-cover` (the image goes from fully covered → fully visible as `correctAnswers` increases from 0 → 10)

**`style.css`** drives animations: `pulse` on correct answers, `shake` on wrong answers, `fadeInOut` during image loading. The `.animal-cover` overlay uses `clip-path: inset(X% 0 0 0)` — JavaScript sets this percentage to control reveal from bottom to top.

**`index.html`** is minimal markup — the numbers grid is entirely generated by JavaScript.

## Key Conventions

- **Language**: All user-facing strings are Dutch and hardcoded directly in `script.js` (not internationalized).
- **Images**: Animals come from Unsplash CDN URLs at 600×600. Each animal object has `{ name, emoji, imageUrl }`. Emoji is the fallback if the image fails to load.
- **Responsive design**: Uses `clamp()` throughout CSS for fluid sizing. Two breakpoints: `max-width: 600px` (mobile) and `max-height: 700px` (short screens). The layout is viewport-constrained — no scrolling is intentional.
- **No dependencies**: Do not introduce npm packages, build tools, or frameworks. Keep this zero-dependency.
